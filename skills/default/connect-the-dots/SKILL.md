# Connect the Dots

Use this skill when information exists, but the structure does not.

## Core Job

Take scattered notes, inputs, or observations and answer:

- where does this belong
- what does it connect to
- is it worth keeping
- what pattern does it reinforce or challenge

## When To Use

- adding a new note into a knowledge graph
- connecting a new research item to existing themes
- updating an index, map, or hub note
- deciding whether something should be promoted into curated memory

## Workflow

1. Read the new material.
2. Determine whether it is worth keeping.
3. Identify the most relevant existing notes or categories.
4. Add links, tags, or placement.
5. Summarize the connection in one or two useful sentences.
6. If a broader pattern emerges, write that pattern down explicitly.

## Quality Rules

- no orphan notes
- no duplicate weak notes when one strong note is enough
- no promotion of low-value material into curated memory
- prefer compression and connection over accumulation

## Output Standard

A good result should make the system easier to scan, not harder.

Prefer:

- one canonical note
- clear links
- a short rationale for why the note matters
