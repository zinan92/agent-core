# Agent Start

This file is for agents, not humans.

If you were given this repository to bootstrap or evaluate, start here.

## Mission

Your job is not to explore the whole repository randomly.

Your job is to:

1. understand the core model
2. scaffold one working demo loop
3. validate that the loop works

## Read Order

Read these in order:

1. `/README.md`
2. `/ARCHITECTURE.md`
3. `/docs/agent/CONTRACT.md`
4. `/docs/agent/BOOTSTRAP.md`

## Primary Rule

Do not treat this repository as a full internal operating environment.

Treat it as a public starter package.

## What You Should Optimize For

- clear boundaries
- reproducible setup
- successful bootstrap
- minimal clean loop

## What You Should Not Do

- invent missing internal history
- create random extra abstractions
- treat the knowledge graph as raw storage
- turn runtime state into source of truth
- add domain-specific complexity before the demo loop works
